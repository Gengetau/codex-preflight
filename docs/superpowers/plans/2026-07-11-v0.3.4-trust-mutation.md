# v0.3.4 Trust Mutation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add default-off, confirmation-gated `trust_approve` and `trust_revoke` MCP tools without changing MCP preflight trust consumption or remote-scan authority.

**Architecture:** Keep confirmation, audit/recovery, core trust persistence, target derivation, and FastMCP transport registration in separate modules. A mutation service owns validation and orchestration; the core trust cache owns locked atomic entry changes; a dedicated HMAC-chained audit supplies write-ahead prepare/commit hooks and startup reconciliation.

**Tech Stack:** Python 3.12, FastMCP, Pydantic/JSON Schema, pytest, Ruff, HMAC-SHA256, existing `locked_cache_file` and atomic JSON primitives.

## Global Constraints

- Base is exactly `33b82fbd0b090ed682edd4b2e36163555c9953ee`; target branch is `codex/v0.3.4-trust-mutation`.
- Only exact `CODEX_PREFLIGHT_ENABLE_TRUST_MUTATION=1` registers `trust_approve` and `trust_revoke`; all eight optional-flag inventories are exact.
- Every confirmed operation uses a process-local 256-bit HMAC key, a 300-second single-use token, a maximum of 128 live challenges, and a 32-per-60-second issuance limit.
- `trust_approve` derives identity, head, fingerprint, scope, policy, and ruleset server-side and never executes the requested command or repository code.
- `trust_revoke` accepts only canonical UUIDv4 entry IDs and exact integer `expectedVersion: 1`.
- Confirmed retries consume an authentic live token before validating the retried envelope or revalidating target state.
- Mutation audit is separate, owner-only, bounded to 4096-byte records, one 1 MiB active segment plus three rotated segments, HMAC chained, fsynced, and fail closed.
- MCP `preflight_check` remains `allow_trust=False`; remote confirmation cannot create or satisfy trust.
- Runtime identity remains exact stdio/unavailable with null client/session IDs; no identity may be inferred.
- Version sources, plugin copies, docs, and examples end at `0.3.4`; normal tests remain local and deterministic.

- Create focused tests in `tests/test_mcp_trust_mutation_confirmation.py`, `tests/test_mcp_trust_mutation_audit.py`, `tests/test_mcp_trust_mutation.py`, and `tests/test_mcp_trust_mutation_registration.py`.
