# Codex Preflight 0.1.x Stabilization Summary

## Status

The 0.1.x line establishes Codex Preflight as a local-first static pre-execution scanner, Python
CLI, Codex plugin package, Codex marketplace package, and minimal read-only MCP-facing runtime.

The line is stable enough to use as a local command gate for Codex-style coding agents, provided
users understand that the scanner is static, heuristic, and best-effort. It does not prove a
repository is safe.

## Completed Areas

### CLI

- Local repository preflight.
- External GitHub repository scan via CLI `--repo`.
- JSON and Markdown reports.
- `exec` wrapper for command gating.
- Trust and cache management.
- Synthetic corpus scanning.

### Safety Analysis

- Planned-command self-risk analysis.
- Static scanner rules.
- Reachability execution graph.
- Capability detection for Node.js, Python, shell, and Docker.
- Conservative uncertainty policy.
- Explicit policy matrix and decision calibration.

### Reachability

- Shell `-c` command forms.
- Python and Node interpreter flags.
- Environment wrappers.
- Package-manager wrappers.
- First-pass Windows command forms.
- Node.js local `require()` and `import` chains.
- Dynamic module uncertainty.

### Packaging

- Codex skill-based plugin package.
- Codex marketplace wrapper.
- Marketplace plugin copy sync automation.
- Plugin and marketplace packaging tests.

### MCP Readiness

- Evidence trust-boundary metadata.
- Cache and trust file locking.
- Read-only local-path-only MCP-facing package.
- Optional MCP runtime smoke coverage.

## Stable Interfaces

- `codex-preflight preflight`
- `codex-preflight exec`
- `codex-preflight corpus scan`
- `codex-preflight rules list`
- `codex-preflight trust ...`
- `codex-preflight cache clear`
- `codex-preflight-mcp --list-tools`
- `codex_preflight_core.preflight.run_preflight`

These interfaces are suitable to depend on within the 0.1.x constraints. Report schemas and
policy decisions may still evolve before a broader 1.0 compatibility promise.

## Intentional Non-Inclusions

- No command execution through MCP.
- No remote repository scanning through MCP.
- No trust approval or revoke through MCP.
- No fake MCP, App, or hook declarations in plugin manifests.
- No hosted service.
- No full JavaScript parser.
- No full Python import graph.
- No dependency vulnerability audit.
- No malware sandbox.

## Known Limitations

- Static analysis is heuristic and best-effort.
- Dynamic runtime behavior can evade static analysis.
- Node.js module resolution is conservative and not a full Node resolver.
- Python import graph support remains limited.
- Shell parsing is intentionally bounded and may miss complex generated command flows.
- External repository scanning is CLI-only.
- MCP integration is intentionally minimal, read-only, and local-path-only.
- Codex Preflight does not replace SAST, dependency audit tools, malware sandboxes, or CVE scanners.

## Recommended Next Line

`v0.2.0` should focus on stabilizing MCP integration and runtime ergonomics without expanding the
tool boundary prematurely. Remote repository MCP scanning, trust mutation, command execution tools,
and deeper language graph support should remain separate, explicitly reviewed milestones.
