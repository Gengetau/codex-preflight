# 0.2.x Version Plan

## v0.2.0

`v0.2.0` adds static README link-poisoning detection. The scanner parses repository-local README
and bounded documentation text for suspicious release/download/install links, raw source archive
downloads, and security-warning bypass wording.

The release is intentionally static-only. It does not fetch linked pages, download artifacts,
open archives, execute installers or scripts, use browser automation, recursively scan linked
repositories, call GitHub metadata APIs, or add MCP tools.

## README Rules

- `README_FAKE_RELEASE_LINK`
- `README_INSTALLER_FROM_NON_RELEASE_HOST`
- `README_RAW_SOURCE_ARCHIVE_DOWNLOAD`
- `README_DEFEAT_SECURITY_WARNING`

Safe read-only commands produce `WARN` for these findings. Install, build, and script-execution
commands produce at least `ASK_USER`.

## v0.2.1+

Later 0.2.x work should return to MCP runtime, schema, UX, and integration documentation
stabilization while preserving the local-path-only, read-only MCP boundary unless a separate loop
explicitly expands it.
